## 进程详解：从概念到实践

### 1. 进程基础概念

#### 1.1 什么是进程

进程是操作系统进行**资源分配和调度的基本单位**，每个进程拥有独立的内存空间，包括代码段、数据段、堆栈段等。进程间内存空间相互隔离，通信需要借助特殊的进程间通信（IPC）机制。

| 特性 | 优势 | 劣势 |
|------|------|------|
| **编程复杂度** | 相互独立，无需考虑复杂同步 | 通信需要特殊机制 |
| **稳定性** | 一个进程崩溃不影响其他进程 | - |
| **资源开销** | 天然避免数据竞争 | 创建和切换成本高 |
| **内存使用** | 资源隔离性好 | 每个进程需要独立地址空间 |

#### 1.2 进程标识符体系

| 标识符 | 全称 | 作用 | 示例 |
|--------|------|------|------|
| **PID** | Process ID | 唯一标识进程 | 每个进程的唯一编号 |
| **PPID** | Parent Process ID | 标识父进程 | 指向创建此进程的父进程 |
| **PGID** | Process Group ID | 标识进程组 | 管道中的多个命令 |
| **SID** | Session ID | 标识会话 | 终端启动的所有进程 |

**实例**：运行 `ls | grep txt` 时，`ls` 和 `grep` 属于同一进程组，共享 PGID 和 SID。

#### 1.3 进程的组成结构

一个完整进程由以下四个核心组件构成：

##### 1.3.1 进程控制块（PCB）
PCB 是进程的"身份证"，包含：
- **进程描述信息**：PID、PPID、PGID、SID
- **进程状态信息**：运行态、就绪态、阻塞态、优先级
- **资源分配清单**：文件描述符、内存页表、信号量
- **CPU 上下文**：寄存器状态、程序计数器、栈指针

##### 1.3.2 其他组成部分
- **代码段**：程序指令序列
- **数据段**：全局变量、静态变量
- **堆栈段**：函数调用栈、局部变量

#### 1.4 特殊进程类型

##### 1.4.1 守护进程 vs 后台进程

| 特性 | 守护进程 | 后台进程 |
|------|----------|----------|
| **定义** | 系统级后台服务进程 | 用户级后台任务 |
| **生命周期** | 系统启动到关闭 | 完成特定任务后退出 |
| **终端依赖** | 完全脱离终端 | 可能仍与终端关联 |
| **启动方式** | 系统自动启动 | 用户手动启动（`command &`） |
| **典型示例** | nginx、mysql、sshd | `./script.sh &` |

**守护进程创建标准流程**：
```cpp
void create_daemon() {
    // 1. 第一次 fork，脱离父进程
    if (fork() > 0) exit(0);
    
    // 2. 创建新会话，脱离控制终端
    setsid();
    
    // 3. 第二次 fork，防止重新获得控制终端
    if (fork() > 0) exit(0);
    
    // 4. 切换工作目录到根目录
    chdir("/");
    
    // 5. 关闭继承的文件描述符
    for (int i = 0; i < 64; ++i) close(i);
    
    // 6. 重定向标准 I/O 到 /dev/null
    open("/dev/null", O_RDONLY);  // stdin
    open("/dev/null", O_RDWR);    // stdout  
    open("/dev/null", O_RDWR);    // stderr
}
```

##### 1.4.2 孤儿进程 vs 僵尸进程

| 类型 | 定义 | 特点 | 危害程度 | 处理方式 |
|------|------|------|----------|----------|
| **孤儿进程** | 父进程先于子进程退出 | 被 init 进程收养 | 无害 | 系统自动处理 |
| **僵尸进程** | 子进程退出但父进程未回收 | 保留 PCB，状态为 Z | 占用 PID 资源 | 父进程主动回收 |

**僵尸进程处理方法**：
```cpp
#include <signal.h>
#include <sys/wait.h>

// 方法1：阻塞等待
int status;
wait(&status);

// 方法2：信号处理（推荐）
void handle_sigchld(int sig) {
    while (waitpid(-1, NULL, WNOHANG) > 0);  // 非阻塞回收所有子进程
}
signal(SIGCHLD, handle_sigchld);
```

#### 1.5 进程调度

##### 1.5.1 进程切换机制

**触发条件**：
- **主动让出**：`sleep()`、`wait()`、`yield()`
- **被动剥夺**：时间片耗尽、高优先级抢占、中断

**切换步骤**：
1. 保存当前进程上下文到 PCB
2. 更新进程状态和队列位置  
3. 选择下一个运行的进程
4. 恢复目标进程的上下文
5. 切换地址空间（更新页表）

##### 1.5.2 调度算法对比

| 算法 | 抢占性 | 特点 | 优势 | 劣势 | 适用场景 |
|------|--------|------|------|------|----------|
| **FCFS** | ✗ | 先到先服务 | 实现简单 | 长作业阻塞短作业 | 批处理系统 |
| **SJF** | ✗ | 最短作业优先 | 平均等待时间短 | 长作业可能饿死 | 已知运行时间 |
| **SRTF** | ✓ | 最短剩余时间 | 响应时间快 | 频繁切换开销大 | 交互式系统 |
| **时间片轮转** | ✓ | 公平分配时间片 | 响应及时，公平性好 | 上下文切换开销 | 分时系统 |
| **多级反馈队列** | ✓ | 多级队列+动态调整 | 综合性能好 | 实现复杂 | 通用操作系统 |

### 2. 进程创建机制

#### 2.1 进程创建流程

1. **复制父进程资源**：采用写时复制（COW）机制，提高效率
2. **分配唯一 PID**：内核分配进程标识符
3. **执行新程序**：可选，使用 `exec` 系列函数
4. **进入调度队列**：等待 CPU 调度

#### 2.2 系统调用对比

| 函数 | 地址空间共享 | 执行顺序 | 特点 | 使用场景 |
|------|-------------|----------|------|----------|
| **fork()** | 否（写时复制） | 并发执行 | 标准进程创建，安全性高 | 通用进程创建 |
| **vfork()** | 是（完全共享） | 子进程先执行 | 高效但危险 | 创建后立即 exec |
| **clone()** | 可配置 | 并发执行 | 精细控制，Linux 特有 | 线程创建，容器技术 |

#### 2.3 exec 系列函数

| 函数 | 参数形式 | 环境变量 | 特点 |
|------|----------|----------|------|
| **execl** | 可变参数列表 | 继承 | 简单直观 |
| **execv** | 参数数组 | 继承 | 动态参数构建 |
| **execle** | 可变参数 + 环境 | 自定义 | 控制环境变量 |
| **execve** | 数组 + 环境 | 自定义 | 系统调用原型 |

**使用示例**：
```cpp
void exec_example() {
    pid_t pid = fork();
    
    if (pid == 0) {
        // 子进程执行新程序
        execl("/bin/ls", "ls", "-l", NULL);
        perror("execl failed");  // 只有失败才会执行到这里
        exit(1);
    } else if (pid > 0) {
        wait(NULL);  // 父进程等待子进程完成
    } else {
        perror("fork failed");
    }
}
```

### 3. 进程间通信（IPC）

#### 3.1 IPC 机制总览

| 机制 | 数据传输方式 | 实现复杂度 | 性能 | 适用场景 |
|------|-------------|------------|------|----------|
| **管道** | 字节流 | 简单 | 中等 | 简单数据传输，父子进程 |
| **消息队列** | 结构化消息 | 中等 | 中等 | 异步消息传递 |
| **共享内存** | 直接内存访问 | 复杂 | 最高 | 大量数据交换 |
| **信号量** | 无数据传输 | 中等 | 高 | 同步与互斥控制 |
| **信号** | 简单通知 | 简单 | 高 | 异步事件通知 |
| **套接字** | 字节流/数据报 | 复杂 | 中等 | 网络通信，跨主机 |

#### 3.2 管道通信

##### 3.2.1 管道特性
- **半双工通信**：数据单向流动
- **先进先出**：队列式数据传递
- **内核缓冲区**：数据读取后即删除
- **阻塞机制**：缓冲区满时写阻塞，空时读阻塞

##### 3.2.2 管道类型对比

| 类型 | 创建方式 | 使用范围 | 生命周期 | 特点 |
|------|----------|----------|----------|------|
| **匿名管道** | `pipe()` | 亲缘进程 | 进程结束时销毁 | 简单，仅限父子进程 |
| **命名管道** | `mkfifo()` | 任意进程 | 文件系统中持久 | 灵活，支持无关进程通信 |

**使用示例**：
```cpp
// 匿名管道示例
int pipefd[2];
pipe(pipefd);  // pipefd[0] 读端，pipefd[1] 写端

if (fork() == 0) {
    // 子进程写数据
    close(pipefd[0]);
    write(pipefd[1], "Hello", 5);
    close(pipefd[1]);
} else {
    // 父进程读数据
    close(pipefd[1]);
    char buffer[10];
    read(pipefd[0], buffer, 5);
    close(pipefd[0]);
}
```

#### 3.3 消息队列

##### 3.3.1 核心特性
- **面向记录**：每条消息是完整单元
- **类型选择**：可按消息类型读取
- **独立生命周期**：进程退出后仍存在
- **异步通信**：发送方和接收方无需同步

##### 3.3.2 主要接口

```cpp
#include <sys/msg.h>

// 创建或打开消息队列
int msgget(key_t key, int msgflg);

// 发送消息
int msgsnd(int msqid, const void *msgp, size_t msgsz, int msgflg);

// 接收消息  
int msgrcv(int msqid, void *msgp, size_t msgsz, long msgtyp, int msgflg);

// 控制消息队列
int msgctl(int msqid, int cmd, struct msqid_ds *buf);
```

**消息类型读取规则**：
- `msgtyp == 0`：读取队列中第一条消息（FIFO）
- `msgtyp > 0`：读取指定类型的第一条消息
- `msgtyp < 0`：读取类型值 ≤ |msgtyp| 中最小的一条

#### 3.4 信号机制

##### 3.4.1 信号来源
1. **硬件来源**：键盘中断（Ctrl+C 产生 SIGINT）
2. **软件来源**：`kill()`、`raise()`、`alarm()` 等系统调用
3. **异常产生**：段错误、除零错误等

##### 3.4.2 信号处理方式
1. **默认处理**：系统预定义行为
2. **自定义处理**：注册信号处理函数
3. **忽略信号**：对非关键信号不做处理

⚠️ **不可处理的信号**：`SIGKILL(9)` 和 `SIGSTOP(19)` 无法被捕获、忽略或阻塞

##### 3.4.3 常用信号表

| 信号名 | 编号 | 默认行为 | 触发条件 | 可否捕获 |
|--------|------|----------|----------|----------|
| **SIGHUP** | 1 | 终止 | 终端挂断 | ✓ |
| **SIGINT** | 2 | 终止 | Ctrl+C | ✓ |
| **SIGKILL** | 9 | 终止 | 强制杀死 | ✗ |
| **SIGSEGV** | 11 | 终止+core | 段错误 | ✓ |
| **SIGTERM** | 15 | 终止 | 请求终止 | ✓ |
| **SIGSTOP** | 19 | 暂停 | 强制暂停 | ✗ |

#### 3.5 信号量同步

##### 3.5.1 核心概念
信号量是用于**进程同步和互斥**的计数器，不传输数据，只协调资源访问。

**基本操作**：
- **P操作（等待）**：信号量值减1，若结果<0则阻塞
- **V操作（释放）**：信号量值加1，若有等待进程则唤醒

##### 3.5.2 应用场景
- **互斥锁**：初值为1，保证同时只有一个进程访问临界区
- **资源计数**：初值为资源总数，控制有限资源的并发访问
- **生产者-消费者**：配合共享内存实现同步

**接口函数**：
```cpp
#include <sys/sem.h>

// 创建或获取信号量集
int semget(key_t key, int num_sems, int sem_flags);

// 信号量操作
int semop(int semid, struct sembuf semoparray[], size_t numops);

// 控制信号量
int semctl(int semid, int sem_num, int cmd, ...);
```

#### 3.6 共享内存

##### 3.6.1 性能优势
共享内存是**最快的IPC方式**，因为：
- 数据无需在内核和用户空间间拷贝
- 多进程直接访问同一物理内存
- 避免了数据序列化/反序列化开销

##### 3.6.2 使用流程
```cpp
#include <sys/shm.h>

// 1. 创建共享内存
int shmid = shmget(IPC_PRIVATE, 1024, IPC_CREAT | 0666);

// 2. 映射到进程地址空间
char *ptr = (char *)shmat(shmid, NULL, 0);

// 3. 读写数据（像普通指针一样）
strcpy(ptr, "Hello Shared Memory");

// 4. 解除映射
shmdt(ptr);

// 5. 删除共享内存
shmctl(shmid, IPC_RMID, NULL);
```

##### 3.6.3 注意事项
- **同步控制**：需要配合信号量等机制保证并发安全
- **资源管理**：必须显式删除，否则系统重启前一直存在
- **权限控制**：注意设置合适的访问权限

### 4. 总结与最佳实践

#### 4.1 IPC 选择指南

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| **简单数据传递** | 管道 | 实现简单，适合父子进程 |
| **异步消息通信** | 消息队列 | 支持消息类型，生命周期独立 |
| **大量数据交换** | 共享内存+信号量 | 性能最优，需要同步控制 |
| **事件通知** | 信号 | 轻量级，异步处理 |
| **跨网络通信** | 套接字 | 支持本地和远程通信 |

#### 4.2 编程注意事项

1. **资源清理**：及时回收子进程，删除IPC对象
2. **错误处理**：检查系统调用返回值，合理处理错误
3. **同步控制**：使用信号量等机制避免竞态条件
4. **性能优化**：根据数据量和实时性要求选择合适的IPC方式

#### 4.3 调试技巧

- 使用 `ps aux` 查看进程状态
- 使用 `ipcs` 查看系统IPC对象
- 使用 `strace` 跟踪系统调用
- 合理使用日志记录关键操作
